Read `get-shit-done/workflows/execute-phase.md`. Add a new step `<step name="hierarchy_dispatch">` BETWEEN the `discover_and_group_plans` step and the `execute_waves` step. Also update the `initialize` step to parse `hierarchy_enabled` and `hierarchy_max_l2_agents` from the INIT JSON (add them to the "Parse JSON for:" instruction list).

The new step must implement the following logic:

**Condition gate:**
```
IF PARALLELIZATION == true AND HIERARCHY_ENABLED == true:
  → hierarchy path (steps 1-6 below)
ELSE:
  → skip to execute_waves (unchanged flat path)
```

**Step 1 — Spawn partitioner to get stream map:**
- Spawn `gsd-partitioner` agent via Task() with `<phase_dir>{phase_dir}</phase_dir>` in prompt
- Parse the returned JSON for `streams` array
- If partitioner fails or returns empty streams or exactly 1 stream: log "Hierarchy: {reason} — using flat execution mode" and skip to execute_waves (this is a Claude's Discretion optimization per research open question #2)

**Step 2 — Create worktrees (track created list for cleanup):**
- For each stream in partition output:
  ```bash
  node "$HOME/.claude/get-shit-done/bin/gsd-tools.cjs" worktree-create {stream.name}
  ```
- Parse returned JSON for `path` and `branch` fields
- Convert relative `path` to absolute: `PROJECT_ROOT=$(git rev-parse --show-toplevel)`, then `{PROJECT_ROOT}/{path}`
- Track all created worktree names in a list (for cleanup on partial failure)
- If ANY worktree-create fails: clean up all already-created worktrees via `worktree-remove {name}`, then fall back to flat execute_waves

**Step 3 — Spawn L2 sub-orchestrators:**
- For each stream, spawn L2 as standalone subagent (NOT TeamCreate — bug #32731):
  ```
  Task(
    subagent_type="gsd-sub-orchestrator",
    run_in_background=true,
    prompt="
      <worktree>{absolute_worktree_path}</worktree>
      <phase>{phase_slug}</phase>
      <phase_dir>{phase_dir}</phase_dir>
      <stream>{ \"name\": \"{stream.name}\", \"plans\": [{plan_list}] }</stream>
    "
  )
  ```
- L1 spawns ALL L2s before waiting for any results (DISP-02: L1 never blocks on a single L2)
- Report what's being dispatched:
  ```
  ## Hierarchy Dispatch
  Spawning {N} L2 sub-orchestrators across {N} worktree streams...
  - Stream {name}: {plan_count} plans → {absolute_worktree_path}
  ```

**Step 4 — Wait for completion and verify:**
- After all background L2 tasks complete, read each return value
- Parse for `STREAM_COMPLETE: {name}` or `STREAM_FAILED: {name}` prefix
- Secondary verification: for each plan in each stream, check SUMMARY.md existence:
  ```bash
  ls {worktree_path}/.planning/phases/{phase_dir}/{phase}-{plan}-SUMMARY.md
  ```
- If ANY L2 returns STREAM_FAILED or SUMMARY.md is missing for a plan that should have completed:
  - Report which stream/plan failed
  - Clean up ALL worktrees (worktree-remove for each)
  - Fall back to flat execute_waves

**Step 5 — Merge worktrees to main:**
- For each stream (in order):
  ```bash
  git merge {worktree_branch} --no-ff -m "merge(hierarchy): stream {stream_name} into main"
  ```
- If merge conflict: notify user "Merge conflict in stream {name} — manual resolution required" and STOP (do NOT fall back to flat — plans already executed in worktrees, flat re-execution would double-run them). This is different from pre-execution failures where flat fallback is safe.

**Step 6 — Reconcile state and cleanup:**
- CRITICAL ORDER: reconcile BEFORE remove (reconcile reads registry, remove clears entries)
  ```bash
  node "$HOME/.claude/get-shit-done/bin/gsd-tools.cjs" state-reconcile
  ```
- Then remove each worktree:
  ```bash
  node "$HOME/.claude/get-shit-done/bin/gsd-tools.cjs" worktree-remove {stream_name}
  ```
- Commit reconciliation:
  ```bash
  node "$HOME/.claude/get-shit-done/bin/gsd-tools.cjs" commit "docs(phase-{X}): reconcile hierarchy execution" --files .planning/STATE.md .planning/worktree-registry.json
  ```
- After successful merge+cleanup: skip execute_waves (plans already executed) and proceed directly to aggregate_results

**Fallback pattern (DISP-06):**
All fallback paths follow the same pattern:
1. Clean up any created worktrees: `worktree-remove {name}` for each in the tracked list
2. Notify user: "Hierarchy execution failed at {step}. Falling back to flat execution mode."
3. Proceed to the existing `execute_waves` step with the same plan inventory

**Anti-patterns to avoid (from research):**
- Do NOT read hierarchy.enabled directly from config.json — use INIT JSON output only
- Do NOT use TeamCreate for L2 spawning — use Task() standalone subagent (bug #32731)
- Do NOT call state-reconcile AFTER worktree-remove — reconcile reads registry that remove clears
- Do NOT implement SendMessage polling — use L2 return text as primary signal
- Do NOT use relative worktree paths in L2 prompt — convert to absolute via git rev-parse --show-toplevel
- Do NOT fall back to flat after merge conflict — plans already executed, flat would double-run

**Format note:** Write the step as workflow Markdown prose (same style as existing steps), not as executable code. The step instructs the orchestrator agent what to do, using bash code blocks for CLI commands.

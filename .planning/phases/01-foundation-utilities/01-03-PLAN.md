  <behavior>
    - FOUND-06a: Given 2 worktrees in registry, reads STATE.md from each worktree's .planning/STATE.md path on disk
    - FOUND-06b: Given main STATE.md with 1 completed plan record and worktree STATE.md with 2 additional records, output has all 3 records
    - FOUND-06c: Given main last_updated="2026-01-01" and worktree last_updated="2026-03-12", output has "2026-03-12" (last-write-wins by timestamp comparison)
    - FOUND-06d: Output STATE.md has exactly one YAML frontmatter block (one pair of --- markers), no duplicate section headers
    - Worktree with missing STATE.md is skipped gracefully
    - Empty worktree registry produces no changes to main STATE.md
  </behavior>
  <action>
    1. In `hierarchy.cjs`, implement `cmdStateReconcile(cwd, raw)`:

    Algorithm:
    a. Read main STATE.md from `path.join(cwd, '.planning', 'STATE.md')`
       - If missing, call `error('STATE.md not found in .planning/')`
    b. Read worktree registry via `readRegistry(cwd)`
       - If no worktrees registered, output "nothing to reconcile" and return
    c. For each active worktree in registry:
       - Compute worktree STATE.md path: `path.join(cwd, entry.path, '.planning', 'STATE.md')`
       - Read via `fs.readFileSync` — skip if missing (log warning)
    d. Parse each STATE.md into two parts:
       - Frontmatter (between --- markers) via `extractFrontmatter()`
       - Body (everything after second ---)
    e. Merge frontmatter (scalar fields — last-write-wins by `last_updated` timestamp):
       - Compare `last_updated` timestamps across main + all worktrees
       - Take the most recent values for: `status`, `stopped_at`, `last_updated`, `last_activity`
       - For `progress` object: take the one with highest `completed_plans` count
    f. Merge body sections:
       - **Performance Metrics table:** Extract table rows (lines starting with `|`), deduplicate by phase name (first column), keep most complete row
       - **Accumulated Context > Decisions:** Collect all `- ` list items under `### Decisions`, deduplicate by content
       - **Accumulated Context > Pending Todos:** Collect all `- ` list items under `### Pending Todos`, deduplicate
       - **Session Continuity:** Last-write-wins (take from the STATE.md with most recent `last_updated`)
    g. Reconstruct STATE.md:
       - Write YAML frontmatter between --- markers
       - Write merged body sections
    h. Write merged STATE.md to main's `.planning/STATE.md`
    i. Call `output()` with merge summary (how many worktrees merged, fields updated)

    2. Key implementation detail: Use `extractFrontmatter()` to separate frontmatter from body. The body starts after the second `---` line. Split the body into sections by `## ` headers. Merge each section independently.

    3. For deduplication of list items: normalize whitespace and compare trimmed strings. This prevents duplicate decisions when two worktrees both logged the same decision.

    4. Handle the edge case where a worktree's STATE.md has additional sections not in main — preserve them by appending.
  </action>

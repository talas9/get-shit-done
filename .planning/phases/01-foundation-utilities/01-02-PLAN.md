    - FOUND-02b: Given plan A (wave 1) and plan B (wave 2, depends_on A), B stays sequential after A within same stream
    - FOUND-02c: Given plan A (wave 1, files: [src/a.ts]) and plan B (wave 1, files: [src/b.ts]), they go in separate streams
    - FOUND-02c-overlap: Given plan A (wave 1, files: [src/shared.ts]) and plan B (wave 1, files: [src/shared.ts]), they go in the SAME stream
    - FOUND-02d: Given 5 wave-1 plans with no overlap and max_l2_agents=2, they are merged into 2 streams
    - Empty phase directory returns { streams: [] }
    - Plan with no files_modified is treated as having no file overlap (can go in any stream)
  </behavior>
  <action>
    1. In `hierarchy.cjs`, implement `cmdHierarchyPartition(cwd, phaseDir, raw)`:

    Algorithm:
    a. Read all files matching `*-PLAN.md` in phaseDir using `fs.readdirSync` + filter
    b. For each file, read content and call `extractFrontmatter()` to get wave, depends_on, files_modified
       - Parse plan ID from filename (e.g., "01-02" from "01-02-PLAN.md")
       - files_modified from frontmatter (array of file paths)
    c. Sort plans by wave (ascending)
    d. Build dependency map: for each plan, record which other plans it depends on
    e. Partition algorithm:
       - Initialize streams array (empty)
       - Process plans wave by wave (ascending)
       - Within each wave, use union-find / disjoint set approach:
         - Two plans in the same wave go in the same stream if they share any file in files_modified
         - Build groups of overlapping plans
       - For plans with depends_on pointing to a plan already assigned to a stream, assign to that same stream (keeps cross-wave deps sequential)
       - For plans with no overlap and no dependency on existing streams, create new stream or assign to least-loaded stream
       - Cap total streams at `max_l2_agents` from `loadConfig(cwd).hierarchy.max_l2_agents`
         - If over cap, merge smallest streams together
    f. Generate stream names: `stream-a`, `stream-b`, etc. (alphabetical)
    g. Output JSON: `{ streams: [{ name: "stream-a", plans: ["01-01", "01-03"], worktree_branch: null }] }`
       - `worktree_branch` is null (set later by worktree-create during dispatch)

    2. Read `max_l2_agents` from config via `loadConfig(cwd)`. Default to 3 if config missing.

    3. Handle edge cases:
       - Empty phase directory: return `{ streams: [] }`
       - Single plan: return one stream with that plan
       - All plans in same wave with no overlap: one stream per plan (up to cap)
       - Plan with missing/empty files_modified: treat as no file overlap

    4. Also provide human-readable output when not `--raw`: print stream assignments as formatted text.
  </action>

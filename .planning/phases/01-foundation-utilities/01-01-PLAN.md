    - cmdWorktreeCreate calls git worktree add with -b flag and the timestamped branch
    - cmdWorktreeCreate registers entry in registry with stream, branch, path, created_at, status fields
    - cmdWorktreeRemove removes worktree directory, deletes branch, removes registry entry
    - cmdWorktreeRemove --force succeeds even when worktree directory does not exist on disk (orphaned cleanup)
    - cmdWorktreeRemove removes registry entry even when git operations fail (self-healing)
  </behavior>
  <action>
    1. In `hierarchy.cjs`, implement `cmdWorktreeCreate(cwd, streamName, raw)`:
       - Generate timestamped branch: `gsd/hierarchy/${timestamp}-${streamName}` where timestamp is ISO with colons/dots replaced by dashes, sliced to 19 chars
       - Compute worktree path: `path.join(cwd, '.claude', 'worktrees', streamName)`
       - Call `execGit(cwd, ['worktree', 'add', worktreePath, '-b', branch])`
       - On failure, call `error()` with stderr message
       - Register in registry via `readRegistry`/`writeRegistry` with fields: `{ stream, branch, path: toPosixPath(relative), created_at, status: 'active' }`
       - Call `output()` with result

    2. Implement `cmdWorktreeRemove(cwd, streamName, force, raw)`:
       - Read registry, find entry by stream name
       - Compute absolute worktree path from entry or from default convention
       - If directory exists (`fs.existsSync`): call `execGit(cwd, ['worktree', 'remove', '--force', absolutePath])`
       - If entry has branch: call `execGit(cwd, ['branch', '-D', entry.branch])` (ignore failures on force)
       - Remove entry from registry array, write back
       - Call `output()` with result

    3. In `gsd-tools.cjs`:
       - Add `require('./lib/hierarchy.cjs')` at the top with the other requires
       - Add switch cases: `case 'worktree-create':` calls `cmdWorktreeCreate(cwd, args[1], raw)`
       - Add `case 'worktree-remove':` calls `cmdWorktreeRemove(cwd, args[1], flags.includes('--force'), raw)`
       - Update the help/usage comment block at the top to include the new commands
